# Creative Production Skill

## 목적

추상적인 전략을 실제 제작자가 사용할 수 있는 구체적인 디자인 산출물로 변환한다.

## 공통 제작 절차

1. 승인된 전략 파일을 확인한다.
2. 결과물의 사용 맥락과 매체를 확인한다.
3. 필수 요구사항과 제약 조건을 추출한다.
4. 콘텐츠 구조를 먼저 설계한다.
5. 시각적 위계와 정보 위계를 정의한다.
6. 시스템과 반복 규칙을 먼저 만든다.
7. 개별 결과물에 시스템을 적용한다.
8. 상태, 예외, 확장 조건을 확인한다.
9. 검수 후 결과물을 저장한다.

## 매체별 필수 섹션 (완성 판정 기준)

아래 섹션이 하나라도 없으면 그 명세는 미완성이다. 자체 검수에서 확인한다.

- 공통: 제작 목표 / 적용 전략 / 제작 명세 / 자체 검수 결과 / 미확정 사항
- BX: 컬러 시스템(hex) / 타이포그래피 / 로고 또는 그래픽 모티프 / 브랜드 애플리케이션 최소 1종
- 그래픽: 레이아웃·그리드 / 색상 / 타이포그래피 / 매체별 사이즈 / (생성형 사용 시) 이미지 생성 프롬프트
- UX/UI: IA 또는 화면 목록 / 화면별 레이아웃 구조 / 컴포넌트·상태 / 디자인 토큰
- 영상: 러닝타임·규격 / 씬별 구성(시간 배분 검산 포함) / 자막·카피 전문 / 편집 가이드

위임 프롬프트에 분량 상한이 명시되면 필수 섹션을 우선 채우고 상한을 지킨다. 상한 안에서는 수치·규칙의 구체성이 산문 설명보다 우선한다.

## 제작 원칙

- 단일 화면보다 시스템을 우선한다.
- 스타일보다 목적과 정보 구조를 우선한다.
- 재사용 가능성과 일관성을 고려한다.
- 디자인 결정마다 근거를 연결한다.
- 정성적 표현을 가능한 한 구체적 규칙으로 변환한다.
- 최종안과 탐색안을 구분한다.
- 생성형 AI 프롬프트는 gx-design 스킬의 PROMPT-PLAYBOOK 규격을 따른다 — 5요소 구조(Subject/Action/Location·Context/Composition/Style), 생성 설정값(종횡비·해상도·길이)의 프롬프트 본문 분리, 컷별 후속 수정 프롬프트 2~3개.

## 프로토타입 제작 원칙

코드를 제작하는 경우 다음을 확인한다.

- 프로젝트의 기존 기술 스택
- 기존 컴포넌트와 스타일
- 반응형 동작
- 키보드 접근성
- 텍스트 대비
- 로딩, 빈 상태, 오류 상태
- 실행 및 빌드 가능 여부
