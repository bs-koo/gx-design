# PRODUCTION-HANDOFF — 실물 제작 게이트

검수 통과본(outputs/final/)은 명세 문서다. 문서는 실물이 아니다 — 영상 명세는 MP4가 아니고, 이미지 명세는 PNG가 아니다.
검수 결과 보고 직후 이 게이트를 연다: **실물 제작 여부와 방식을 AskUserQuestion으로 묻는다.**
묻지 않고 명세만 남긴 채 종료하지 않는다. 묻지 않고 실물 제작을 시작하지도 않는다.
결과물이 문서 자체(브랜드 가이드·전략 문서·카피 등)인 프로젝트는 이 게이트를 생략하고 생략 사실을 보고한다.

## 1. 질문 구성

1문항, header "실물 제작", multiSelect: true(경로 병행 가능), preview는 쓰지 않는다(다중 선택과 호환되지 않음).
결과물 유형별 옵션:

| 결과물 유형 | 옵션 4개 |
|---|---|
| 영상 | 코드 렌더링(Remotion·FFmpeg → 실제 MP4) · Gemini 프롬프트(Veo 계열 영상 생성) · 편집 도구 핸드오프(브리프에서 정한 도구 기준 체크리스트) · 문서로 종료 |
| 이미지·그래픽 | 코드 렌더링(HTML/SVG → PNG) · Gemini 프롬프트(이미지 생성) · .pen 시안(pencil MCP 가용 시, 아니면 디자인 툴 가이드) · 문서로 종료 |
| UI | 프로토타입(frontend-design 스킬) · .pen 시안 · Gemini 프롬프트(비주얼 목업) · 문서로 종료 |

BX처럼 산출물이 혼합(시각 자산 + 가이드 문서)인 프로젝트는 실물화 가능한 시각 자산(로고·키 비주얼·SNS 그래픽 등)에 이미지·그래픽 행을 적용한다 — 순수 문서만 남는 경우가 아니면 게이트를 생략하지 않는다.

**영상·이미지 유형에서 Gemini 옵션은 필수다** — 이 플러그인의 주 사용자층(개발자·사업부 실무자)이 생성 도구로 Gemini를 주로 쓴다. 어떤 경우에도 빼지 않는다.

추천 배치 — 아래 기준으로 고른 추천 경로를 **옵션 목록의 첫 번째 위치로 옮기고** label 끝에 "(추천)"을 붙인다. 표의 나열 순서는 옵션 순서가 아니다. label은 경로명만 짧게 쓰고, 괄호 설명은 description에 넣는다.

- UI·터미널·타이포·도형·차트 중심 산출물 → **코드 렌더링** (텍스트 정확도 100%, 수정 후 재렌더 가능)
- 실사·인물·일러스트·질감 중심 산출물 → **Gemini 프롬프트** (코드로 재현할 수 없는 영역)

사용자 선택이 추천과 달라도 그대로 진행한다. 단, 선택 경로가 산출물의 핵심 요소와 구조적으로 충돌하면(예: 타이포 중심 산출물에 생성형 경로 — 온스크린 텍스트가 전부 후삽입으로 빠진다) 위임 전에 그 한계와 보완책(코드 렌더링 병행, 자막 후삽입 계획 등)을 한 번 고지하고 진행한다. 선택을 번복시키려는 재질문은 하지 않는다.

## 2. 경로 A — 코드 렌더링

영상은 Remotion(React 컴포지션 → MP4) 또는 FFmpeg, 이미지는 HTML/CSS → 스크린샷(Playwright) 또는 SVG로 만든다.

- 명세의 프레임·픽셀·색상 값을 그대로 코드로 옮긴다. 환경 제약으로 바꾼 값(폰트 폴백 등)은 [가정] 표기 후 보고한다.
- 렌더 전후 [VISUAL-QUALITY.md](VISUAL-QUALITY.md)의 시각 품질 루프를 수행한다 — 대표 컷 2~3장을 먼저 렌더해 직접 열어 보고 비평·수정하며(최소 1회·최대 3회), 통과 후 전체 렌더로 간다.
- 실물은 outputs/final/에 저장하고, 렌더 프로젝트는 outputs/video/ 같은 별도 폴더에 남겨 재렌더 가능하게 한다.
- 의존성 설치·렌더 환경 구성은 오케스트레이터가 직접 수행해도 된다(위임 예외).

## 3. 경로 B — Gemini 프롬프트 패키지

creative-producer에게 위임한다. 위임 프롬프트에 final 명세 경로와 [PROMPT-PLAYBOOK.md](PROMPT-PLAYBOOK.md) 경로를 함께 전달하고, 패키지 구조·프롬프트 규격·텍스트 정책은 그 문서를 정본으로 따르게 한다.
저장: `outputs/final/YYYY-MM-DD_<project>_gemini-prompts.md`

핵심 원칙 요약(상세와 예외는 PROMPT-PLAYBOOK):

- 생성 설정값(종횡비·해상도·클립 길이)은 프롬프트 문장이 아니라 UI 설정으로 분리 안내한다.
- 컷마다 후속 수정 프롬프트 2~3개를 포함한다 — 재롤이 아니라 같은 대화의 부분 수정이 기본 경로다.
- 온스크린 텍스트는 PROMPT-PLAYBOOK §4 정책을 따른다(정밀 로고·한글 문장은 후삽입, 짧은 영문 1~10자는 허용).
- 영상은 클립당 8초 한계로 분할하고 모든 클립에 오디오 지시를 넣는다.
- 프롬프트 본문은 영어가 기본(생성 안정성), 사용자가 원하면 한국어로 쓴다.

## 4. 경로 C — 편집 도구 핸드오프

브리프에서 정한 편집 도구(CapCut 등) 기준의 단계별 실행 체크리스트를 정리한다.
production 패키지에 편집 가이드가 이미 있으면 새로 만들지 않고 해당 절의 위치와 남은 작업만 안내한다.

## 5. 문서로 종료

실물을 만들지 않는 선택도 유효하다. 남은 사용자 액션(게시·트래킹 등)을 요약하고 종료한다.

## 완료 기준

- 실물 제작 여부를 물었다(결과물이 문서 자체인 경우: 생략을 보고했다).
- 선택된 각 경로의 산출물이 outputs/final/에 저장되었고 4줄 보고를 마쳤다.
- 복수 경로를 선택받았으면 전부 완료했다.
