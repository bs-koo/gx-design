# 전략 N안 병렬 개발 — Design It Twice

목표: 서로 **전략적으로** 다른 콘셉트를 병렬로 만들고, 비교와 소신 추천을 거쳐 사용자가 선택하게 한다.
"표현만 다른 같은 전략" 3개는 실패다 — 철학이 달라야 한다.

## 절차

1. design-strategist 서브에이전트 3개를 병렬 스폰한다. 전원에게 동일하게: 브리프 경로, 리서치 경로, 위임 11항목.
2. 각자에게 서로 다른 철학을 배정한다(매체에 맞게 문구 조정):
   - **A안 — 본질 우선**: 타깃의 핵심 문제 하나를 가장 적은 요소로 해결하는 방향
   - **B안 — 차별화 우선**: 경쟁 지형의 빈 위치를 점유하는 반대 포지션
   - **C안 — 시스템 우선**: 확장·반복 운영을 전제로 한 플랫폼적 접근
3. 각 안은 design-strategy 스킬의 콘셉트 작성 형식(Concept Name ~ Risks)으로 outputs/strategy/에 저장하게 한다.
4. 세 안을 받으면 비교표를 만든다. 축: 브리프 적합 / 차별성 / 실행 가능성 / 리스크.
5. 소신 추천 1안을 정하고 근거를 쓴다 — 무관심한 나열이 아니라 확신을 제시한다.
   두 안 이상이 사실상 같은 전략이면 해당 안을 다른 철학으로 재작성시킨다.
6. AskUserQuestion으로 선택받는다: 문항 1개, 옵션 = 각 안(추천안이 첫 번째, "(추천)" 표기),
   옵션마다 preview에 Concept Name · One-line Definition · Core Insight 요약을 넣는다.
7. 선택 결과와 근거를 outputs/strategy/에 선택 기록으로 저장하고 단계 3으로 간다.
