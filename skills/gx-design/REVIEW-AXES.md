# 2축 독립 검수

목표: 전략 일치와 완성도를 서로 독립된 눈으로 검수한다.
두 축의 결과는 합치지 않는다 — 한 축의 호평이 다른 축의 결함을 가리지 못하게 한다.

## 절차

1. 검수 서브에이전트 2개를 병렬 스폰한다. 각자 creative-review 스킬을 로드하게 하고, 서로의 존재와 결과를 알리지 않는다.
   - **축 A — 전략 일치**: 입력 = 브리프 + 선택된 전략 + 산출물. 검수 범위 = 목표 적합성, 전략 일치성.
   - **축 B — 완성도**: 입력 = 산출물 + 실물 이미지·렌더 결과 경로(존재 시 필수 — VISUAL-QUALITY.md의 렌더 품질 로그 포함) + 매체 정보. 검수 범위 = 정보 구조, 시각 완성도(실물 이미지를 직접 열어 확인), UX/UI 또는 영상, 제작 가능성.
2. 각 축은 creative-review의 검수 결과 형식(총평 ~ 최종 승인 여부)으로 보고하게 한다.
3. 두 보고를 병기해 사용자에게 전달한다. 병합하거나 재정렬하지 않는다.
4. Critical이 하나라도 있으면: 해당 이슈를 만든 단계의 에이전트에게 수정을 위임하고, 수정 후 **그 축만** 재검수한다.
5. 두 축 모두 Critical 0건이면 통과본을 `outputs/final/YYYY-MM-DD_<project>_final.md`로 저장하고 사용자에게 최종 보고한다.

## 핵심 모드 조항

핵심 모드에서는 검수 서브에이전트 1개가 축 A → 축 B를 순차 수행한다. 두 축의 보고는 여전히 분리 작성하고 합치지 않는다. Critical만 수정 루프를 돌리고(수정 후 해당 축만 재점검), Major 이하는 기록만 남긴다.
