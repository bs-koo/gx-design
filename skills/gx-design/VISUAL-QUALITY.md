# VISUAL-QUALITY — 코드 렌더링 시각 품질 루프

문서 검수만으로는 실물 품질을 보증할 수 없다 — 렌더된 이미지를 직접 열어 보고 비평하는 것이 이 문서의 전부다.
적용 대상: PRODUCTION-HANDOFF 경로 A(코드 렌더링)와 UI 프로토타입 등 이미지·화면 실물이 나오는 모든 제작. 모드(전체/핵심)와 무관하게 의무다.

## 절차

1. **제작 전**: frontend-design 스킬이 가용하면 로드하고, [VISUAL-CRAFT.md](VISUAL-CRAFT.md)의 크래프트 규칙(안티-슬롭·톤 다이얼·일관성·모션)을 제작·비평 기준으로 삼는다.
2. **대표 컷 렌더·캡처**: 전체 렌더 전에 대표 컷(스틸·대표 화면) 2~3장을 먼저 렌더한다. 화면 산출물은 Playwright MCP가 가용하면 뷰포트별로 캡처하고 레퍼런스가 있으면 비교한다 — 없으면 렌더 PNG를 그대로 연다(폴백).
3. **시각 비평(핵심)**: 렌더 이미지를 Read로 직접 열어 보고 아래 기준으로 비평한다. 명세 값 대조가 아니라 디자인 비평이다.
   - **시각 위계**: 시선이 의도한 순서로 흐르는가
   - **간격·정렬**: 리듬이 일관적인가, 어색한 붙음·벌어짐은 없는가
   - **밀도**: 빈약해 허전하거나 과밀하지 않은가
   - **색**: 균형·대비가 충분한가, 팔레트를 벗어난 색은 없는가
   - **정체성 오독**: 다른 것으로 읽히지 않는가 — "이 심볼이 막대 차트로 보이지는 않는가", "이 키 비주얼이 미완성 와이어프레임으로 보이지는 않는가"를 명시적으로 자문한다
   - **QA 4항목**: AI 아티팩트·렌더 결함(폰트 폴백 흔적, 깨짐) / 기술 정확성(치수·해상도·hex) / 구성 문제(잘림·오버랩) / 스타일 일관성(다른 자산과)
   - **크래프트([VISUAL-CRAFT.md](VISUAL-CRAFT.md))**: 안티-슬롭(§1 3대 기본 룩을 무심코 답습하지 않았는가) / 일관성 잠금(§3 accent·radius·테마가 각 하나인가) / 모션(§4 있으면 transform·opacity·나가는 모션<들어오는 모션인가) / 시그니처(§5 대담함이 한 곳에 모였는가)
4. **수정 → 재렌더 → 재비평**: 루프는 **최소 1회, 최대 3회**. 회차마다 비평 요지(발견 → 수정)를 제작 문서의 "렌더 품질 로그" 절에 기록한다.
5. **통과**: 자체 비평 Critical 0건이면 전체 렌더로 진행한다.
6. **3회 초과**: Critical이 남으면 자동 진행하지 않는다 — 사용자에게 한계와 대안(경로 변경·수동 보정·명세 조정)을 보고한다.

## 검수 연계

- 검수(REVIEW-AXES) 축 B의 입력에 실물 이미지 경로와 렌더 품질 로그를 포함한다.
- 루프 기록이 없는 실물은 검수에서 Critical("품질 루프 미수행")로 처리한다.

## Red Flags

- 렌더 파일을 저장만 하고 열어 보지 않았다
- "명세 값과 일치한다"만 확인하고 통과시켰다 — 값이 맞아도 조잡할 수 있다. 비평 기준은 절차 3의 6가지다
- 렌더 품질 로그 없이 outputs/final/에 실물을 저장했다
